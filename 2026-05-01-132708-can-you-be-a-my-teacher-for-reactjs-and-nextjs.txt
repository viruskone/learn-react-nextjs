     Lesson Progress Registry

     A JSON file at todo-app/lesson-progress.json tracks the student's current
     position.

     {
       "currentLesson": 1,
       "lessons": {
         "1":  { "status": "in_progress", "startedAt": "2026-05-01" },
         "2":  { "status": "not_started" }
       }
     }

     Statuses: "not_started" | "in_progress" | "completed"

     This file lives inside todo-app/ so it travels with the project. It is
     committed to git so progress is persisted across sessions.

     New slash command: /start-lesson

     File: .claude/commands/start-lesson.md

     When the student runs /start-lesson [N] (or just /start-lesson to advance
     to next):
     1. Read todo-app/lesson-progress.json
     2. Mark the current lesson as "completed" (if it was "in_progress")
     3. Set the target lesson to "in_progress" with today's date
     4. Write the updated JSON back
     5. Print the lesson title, theory summary, and task from
     lessons/lesson-NN/README.md + task.md

     When the student runs /review-lesson, the grading agent reads currentLesson
      from the registry automatically — no need to ask which lesson.
